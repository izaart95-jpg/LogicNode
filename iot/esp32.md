# ESP32 Documentation

## 1. Origin of ESP32

The **ESP32** is a low-cost, low-power System on Chip (SoC) with integrated Wi-Fi and Bluetooth, developed by **Espressif Systems**.

- **Company:** Espressif Systems (Shanghai, China)
- **First Release:** 2016
- **Predecessor:** ESP8266
- **Architecture:** Xtensa (later variants use RISC-V)

Espressif designed the ESP32 as a more powerful and feature-rich successor to the ESP8266, targeting IoT (Internet of Things), embedded systems, and wireless applications.

---

## 2. ESP32 Architecture Overview

The ESP32 integrates:

- Dual-core or single-core CPU
- Wi-Fi (802.11 b/g/n)
- Bluetooth (Classic + BLE)
- SRAM + external flash support
- Multiple peripheral interfaces
- ADC, DAC, PWM, Touch sensors

### Core Architecture

- 32-bit microcontroller
- Up to 240 MHz clock speed
- Harvard architecture
- Low power modes (Modem sleep, Light sleep, Deep sleep)

---

## 3. ESP32 Variants

Espressif has released multiple ESP32 families:

### 3.1 ESP32 (Original Series)
- Xtensa dual-core LX6
- Wi-Fi + Bluetooth Classic + BLE
- Most widely used

### 3.2 ESP32-S2
- Single-core Xtensa LX7
- Wi-Fi only (no Bluetooth)
- USB OTG support
- Improved security

### 3.3 ESP32-S3
- Dual-core Xtensa LX7
- Wi-Fi + BLE 5
- AI acceleration (vector instructions)
- USB OTG

### 3.4 ESP32-C3
- Single-core RISC-V
- Wi-Fi + BLE 5
- Lower cost
- Improved security features

### 3.5 ESP32-C6
- RISC-V core
- Wi-Fi 6
- BLE 5
- Thread / Zigbee (802.15.4)

### 3.6 ESP32-H2
- RISC-V
- BLE 5
- Zigbee / Thread
- No Wi-Fi

## 4. Key Features

### Wireless Connectivity
- 2.4 GHz Wi-Fi
- Bluetooth Classic
- Bluetooth Low Energy (BLE)
- Some variants support Wi-Fi 6, Zigbee, Thread

### Processing
- Up to dual-core CPU
- Up to 240 MHz
- Hardware floating point (in some models)

### Memory
- Internal SRAM
- External SPI Flash support
- PSRAM support (some variants)

### GPIO & Peripherals
- Up to 34 GPIO pins (varies by model)
- SPI, I2C, I2S
- UART
- CAN (TWAI)
- PWM (LEDC)
- Touch sensors
- Hall sensor (original ESP32)
- SD card interface

### Analog Features
- 12-bit ADC
- 8-bit DAC (original ESP32)
- Capacitive touch sensing

### Security
- Secure Boot
- Flash Encryption
- Hardware crypto accelerators (AES, SHA, RSA, ECC)

## 5. Power Modes

- Active mode
- Modem sleep
- Light sleep
- Deep sleep (Ultra low power)
- ULP co-processor (in some variants)

Deep sleep current can go as low as a few microamps.

- C

### 6.1 Arduino IDE for ESP32

The Arduino IDE provides a beginner-friendly way to program ESP32 boards.

**Board Manager Setup:**
1. Open Arduino IDE → File → Preferences
2. Add to "Additional Board Manager URLs":
   ```
   https://espressif.github.io/arduino-esp32/package_esp32_index.json
   ```
3. Tools → Board → Boards Manager → Search "esp32" → Install "esp32 by Espressif Systems"
4. Select your board: Tools → Board → ESP32 → (your variant)
5. Select COM port: Tools → Port

**Upload Process:**
```
Compile → Hold BOOT button (some boards) → Upload → Release BOOT → Serial Monitor
```

**Common Libraries:**
- WiFi.h (built-in Wi-Fi)
- BluetoothSerial.h (Classic Bluetooth)
- BLE libraries (BLE communication)
- ESP32Servo (servo motor control)
- AsyncTCP + ESPAsyncWebServer (async web server)

**Common Issues:**
- "Failed to connect": Hold BOOT button during upload, or check USB cable (data vs charge-only)
- "No module named serial": Install pyserial (`pip install pyserial`)
- Board not showing in port: Install CP2102 or CH340 USB driver

### 6.2 ESP-IDF (Espressif IoT Development Framework)

ESP-IDF is Espressif's official development framework — more powerful than Arduino but steeper learning curve.

**Installation (Linux/macOS):**
```bash
# Install prerequisites
sudo apt install git wget flex bison gperf python3 python3-venv cmake ninja-build ccache libffi-dev libssl-dev dfu-util libusb-1.0-0

# Clone ESP-IDF
mkdir -p ~/esp && cd ~/esp
git clone --recursive https://github.com/espressif/esp-idf.git
cd esp-idf
./install.sh esp32    # or esp32s3, esp32c3, etc.

# Set up environment (run each session)
. $HOME/esp/esp-idf/export.sh
```

**Project Structure:**
```
my_project/
├── CMakeLists.txt
├── main/
│   ├── CMakeLists.txt
│   └── main.c
├── components/       (custom components)
└── sdkconfig         (generated by menuconfig)
```

**Build, Flash, Monitor:**
```bash
# Create project from template
idf.py create-project my_project && cd my_project

# Configure (menuconfig TUI)
idf.py menuconfig

# Build
idf.py build

# Flash to device
idf.py -p /dev/ttyUSB0 flash

# Serial monitor
idf.py -p /dev/ttyUSB0 monitor

# All-in-one
idf.py -p /dev/ttyUSB0 flash monitor
```

**ESP-IDF vs Arduino:**

| Feature | Arduino IDE | ESP-IDF |
|---------|-------------|---------|
| Learning Curve | Easy | Moderate-Steep |
| Configuration | Limited | Full (menuconfig) |
| RTOS Access | Limited | Full FreeRTOS |
| Performance | Good | Optimal |
| OTA Updates | Basic | Advanced |
| Best For | Prototyping | Production |

### 6.3 PlatformIO

PlatformIO combines the ease of Arduino with the power of ESP-IDF.

- **Install:** VS Code extension "PlatformIO IDE"
- **Features:** Multi-framework (Arduino + ESP-IDF), library manager, unit testing, debugging
- **Config:** `platformio.ini` file per project

```ini
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
monitor_speed = 115200
lib_deps =
    ESP Async WebServer
```

## 8. Comparison with ESP8266

| Feature | ESP8266 | ESP32 |
|----------|----------|--------|
| CPU | Single-core | Single/Dual-core |
| Bluetooth | No | Yes |
| GPIO | Limited | More |
| ADC | 10-bit | 12-bit |
| Performance | Lower | Higher |
| Security | Basic | Advanced |

---

## 9. Advantages

- Very low cost
- Built-in wireless connectivity
- Large community support
- Rich peripheral support
- Strong power management
- Secure hardware features

---

## 10. Limitations

- 2.4 GHz only (most variants)
- ADC noise issues (original ESP32)
- Wi-Fi can increase power consumption
- Limited RAM compared to larger MCUs
